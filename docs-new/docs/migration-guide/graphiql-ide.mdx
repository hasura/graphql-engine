# GraphiQL IDE

We use graphiql editor wherever applicable for example showcase throughout docs. Syntax migration is staright forward for this.
A GraphiQLIDE component is already created with required custom logic. So, use `GraphiQLIDE` just like any other react component.

## Sphinx - RST

```rst
// highlight-start
.. graphiql::
  :view_only:
  :query:
// highlight-end
    query get_authors_in_pincode ($jsonFilter: jsonb){
      authors(
        where: {
          address: {_contains: $jsonFilter }
        }
      ) {
        id
        name
        address
      }
    }
  // highlight-next-line
  :response:
    {
      "data": {
        "authors": [
          {
            "id": 1,
            "name": "Ash",
            "address": {
              "street_address": "161, 19th Main Road, Koramangala 6th Block",
              "city": "Bengaluru",
              "state": "Karnataka",
              "pincode": 560095,
              "phone": "9090909090",
            }
          }
        ]
      }
    }
  // highlight-next-line
  :variables:
    {
      "jsonFilter": {
        "pincode": 560095
      }
    }
```

:::note
  `:view_only:` can be ignored as all the existing snippets have this.

  So, the prop in new react component is set to true by default. In case, any snippet doesn't include view-only. Just send a false for this. `viewOnly={false}` to `GraphQLIDE`.
:::
